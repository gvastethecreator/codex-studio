# Style Preset Card Regeneration Backlog

## Criterio

Las default cards de presets se generan como `.webp` en `assets/recipes/styles/defaults/`.
`scripts/generate-style-defaults.ts` construye el prompt desde el pack, la categorĂ­a, el `name`, el `visualDna`, el `negativePrompt` y variantes determinĂ­sticas por hash.

El prompt exacto usado para una card no queda versionado junto al asset. Los manifests `assets/recipes/styles/defaults/manifest-<pack>.json` guardan `presetId`, `presetName`, `jobId`, modelo, modo y timestamp, pero no el prompt completo.

Regla operativa:

- Si cambia `name`, `visualDna`, `avoidRules` o `attributes.negativePrompt`, la card existente se considera obsoleta.
- Todo preset modificado debe anotarse aquĂ­ con estado `needs-regeneration`.
- Al regenerar, reemplazar `assets/recipes/styles/defaults/<PRESET_ID>.webp` y actualizar el checkpoint `manifest-<pack>.json`.
- No cerrar una tanda visual como final si quedan cards obsoletas sin anotaciĂłn.

## Pendiente de regeneraciĂłn

Estado: `needs-regeneration`
Motivo comĂşn: se reescribiĂł el preset desde escena/props concretos hacia estilo abstracto aplicable a cualquier prompt o imagen de entrada.

Nota de criterio: tĂ­tulos, IPs o nombres de obra pueden mantenerse si funcionan como ancla estilĂ­stica. La card debe regenerarse igual cuando cambia el manifest, pero el problema no es la IP en sĂ­; el problema es que `visualDna` o `creative_brief` fuercen composiciĂłn, personaje, locaciĂłn, prop o evento especĂ­fico.
